# Headlamp Kubernetes UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Headlamp as a Kubernetes web UI accessible at `headlamp.homeserver` via Traefik Ingress, deployed through the existing ArgoCD GitOps pipeline.

**Architecture:** Wrapper Helm chart in `argocd/apps/headlamp/` with the official `headlamp-k8s/headlamp` chart as a dependency. Two custom templates handle cluster-admin RBAC and a long-lived ServiceAccount token. The existing ApplicationSet picks up the new directory automatically — no ArgoCD config changes needed.

**Tech Stack:** Helm 3, ArgoCD ApplicationSet, Traefik Ingress, Kubernetes RBAC

### Task 2: values.yaml

**Files:**
- Create: `argocd/apps/headlamp/values.yaml`

- [ ] **Create values with Traefik ingress and resource limits**

```yaml
---
# Values passed through to the headlamp sub-chart.
# All keys live under `headlamp:` because that is the dependency name.
headlamp:
  replicaCount: 1

  ingress:
    enabled: true
    ingressClassName: traefik
    annotations: {}
    hosts:
      - host: headlamp.homeserver
        paths:
          - path: /
            pathType: Prefix
    tls: []

  resources:
    requests:
      cpu: 50m
      memory: 64Mi
    limits:
      cpu: 200m
      memory: 128Mi

  # The upstream chart creates a ServiceAccount named after the release.
  # We bind it to cluster-admin in templates/clusterrolebinding.yaml.
  serviceAccount:
    create: true
```

- [ ] **Verify rendering locally** (requires `helm` CLI)

```bash
helm repo add headlamp https://headlamp-k8s.github.io/headlamp/
helm dep build argocd/apps/headlamp
helm template headlamp argocd/apps/headlamp --namespace headlamp | grep -E 'kind:|name:'
```

Expected output contains: `kind: Deployment`, `kind: Service`, `kind: Ingress`, `kind: ServiceAccount`

- [ ] **Commit**

```bash
git add argocd/apps/headlamp/values.yaml
git commit -m "feat(headlamp): add values with Traefik ingress and resource limits"
```

### Task 3: ClusterRoleBinding template

**Files:**
- Create: `argocd/apps/headlamp/templates/clusterrolebinding.yaml`

- [ ] **Create the ClusterRoleBinding**

```yaml
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: {{ .Release.Name }}-cluster-admin
  labels:
    app.kubernetes.io/managed-by: {{ .Release.Service }}
    helm.sh/chart: {{ .Chart.Name }}-{{ .Chart.Version }}
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: cluster-admin
subjects:
  - kind: ServiceAccount
    name: {{ .Release.Name }}
    namespace: {{ .Release.Namespace }}
```

- [ ] **Verify it renders correctly**

```bash
helm template headlamp argocd/apps/headlamp --namespace headlamp | grep -A 12 'ClusterRoleBinding'
```

Expected: `name: headlamp-cluster-admin`, subjects referencing `headlamp` SA in `headlamp` namespace.

- [ ] **Commit**

```bash
git add argocd/apps/headlamp/templates/clusterrolebinding.yaml
git commit -m "feat(headlamp): bind headlamp ServiceAccount to cluster-admin"
```

### Task 4: Long-lived token Secret

**Files:**
- Create: `argocd/apps/headlamp/templates/token-secret.yaml`

- [ ] **Create the ServiceAccount token Secret**

```yaml
---
apiVersion: v1
kind: Secret
metadata:
  name: headlamp-admin-token
  namespace: {{ .Release.Namespace }}
  annotations:
    kubernetes.io/service-account.name: {{ .Release.Name }}
type: kubernetes.io/service-account-token
```

Kubernetes automatically populates `data.token` once the Secret is created and the SA exists.

- [ ] **Verify it renders correctly**

```bash
helm template headlamp argocd/apps/headlamp --namespace headlamp | grep -A 8 'token-secret\|service-account-token'
```

Expected: Secret with annotation `kubernetes.io/service-account.name: headlamp` and `type: kubernetes.io/service-account-token`.

- [ ] **Run helm lint to catch any issues**

```bash
helm lint argocd/apps/headlamp
```

Expected: `1 chart(s) linted, 0 chart(s) failed`

- [ ] **Commit**

```bash
git add argocd/apps/headlamp/templates/token-secret.yaml
git commit -m "feat(headlamp): add long-lived ServiceAccount token Secret"
```

### Task 6: Retrieve admin token and log in

- [ ] **Get the admin token**

```bash

  -o jsonpath='{.data.token}' | base64 -d
```

Copy the printed token.

- [ ] **Add a local hosts entry** (on your dev machine, not the server) so `headlamp.homeserver` resolves:

```bash
# Linux/macOS — add to /etc/hosts:
192.168.178.127  headlamp.homeserver
```

- [ ] **Open Headlamp in the browser**

Navigate to `http://headlamp.homeserver`, paste the token from the step above.

Expected: Headlamp dashboard showing the k3s cluster nodes and namespaces.
